# Переезд с Netlify на Vercel

## Что изменилось технически
- `netlify/functions/chat.js` (формат Netlify) → `api/chat.js` (формат Vercel)
- `netlify.toml` больше не нужен — Vercel сам роутит `/api/chat.js` в `/api/chat`, без конфига
- Добавлен `vercel.json` — поднимает лимит времени функции до 60 секунд
  (у Netlify на бесплатном тарифе было жёстко 10 сек, у Vercel Hobby можно до 60 — это
  само по себе решает часть проблем с таймаутами, независимо от скорости модели)
- `index.html` не менялся — он уже стучится в `/api/chat`, это совпадает с роутингом Vercel

Старые файлы `netlify.toml` и папку `netlify/` можно не удалять — Vercel их просто
проигнорирует, они не мешают. Но если хочешь порядка в репозитории — можно снести.

## Шаги

### 1. Залей обновлённые файлы в тот же GitHub-репозиторий
Добавь/замени:
- `api/chat.js`
- `vercel.json`
- `package.json`
- `index.html` (без изменений, но убедись, что он последней версии)

### 2. Импортируй проект в Vercel
1. https://vercel.com → Sign up / Login (проще через GitHub-аккаунт)
2. **Add New... → Project → Import Git Repository** → выбери свой репозиторий
3. Framework Preset — оставь **Other** (не Next.js), Build command и Output directory
   оставь пустыми — это чисто статический `index.html` + одна серверная функция

### 3. Добавь переменные окружения
**Перед первым деплоем** (или после — потом просто передеплой):
Project → **Settings → Environment Variables**, добавь:
- `OPENROUTER_API_KEY` — твой ключ с openrouter.ai
- `OPENROUTER_MODEL` — id модели (необязательно, без неё используется дефолтная бесплатная)
- `SUPABASE_URL` — тот же, что в index.html
- `SUPABASE_ANON_KEY` — тот же анонимный ключ

### 4. Deploy
Deployments → на последнем деплое **⋮ → Redeploy** (если добавил переменные после
первого деплоя — обязательно передеплой, иначе новые переменные не подхватятся)

### 5. Проверка
Открой `https://твой-проект.vercel.app/api/chat` в браузере — должен вернуться
`{"error":"Method not allowed"}` (это нормально, значит функция жива, просто ждёт POST-запрос).
Если 404 — значит файл `api/chat.js` не задеплоился, проверь структуру папок в репозитории:
он должен лежать в `api/chat.js` в корне репозитория, не во вложенной подпапке.

## На будущее — не наступить на те же грабли с лимитами
- **Netlify:** бесплатный тариф — 300 «кредитов»/месяц на всё (бандвич + функции), 10 сек
  на функцию. Легко упереться при активном использовании.
- **Vercel Hobby:** бесплатно, но с оговоркой — **только для некоммерческих проектов**
  по правилам самого Vercel. Если сервис уже продаётся клиентам — формально нужен Pro
  ($20/мес), там снимаются ограничения и по деньгам, и по времени функции (до 300 сек).
  Для демо и первых 5-дневных пробников Hobby, скорее всего, отработает без проблем,
  но держи это в голове, когда пойдут первые платящие клиенты.
