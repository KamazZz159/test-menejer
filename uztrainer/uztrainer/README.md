# Тренажёр звонков — версия с безопасным бэкендом (Netlify)

## Что изменилось
Ключ и модель убраны из index.html и переехали в `netlify/functions/chat.js`,
который читает их из переменных окружения. В браузере клиента ключ теперь не виден.

## Деплой
1. Получи новый ключ OpenRouter (старый лучше сразу отозвать на openrouter.ai/keys —
   он был публично виден в предыдущей версии файла и мог утечь)
2. Залей эти файлы в GitHub-репозиторий, сохранив структуру:
   - `index.html`
   - `netlify/functions/chat.js`
   - `netlify.toml`
3. app.netlify.com → Add new site → Import from Git → выбери репозиторий
4. Site configuration → Environment variables → добавь:
   - `OPENROUTER_API_KEY` = твой ключ
   - (необязательно) `OPENROUTER_MODEL` = id платной модели, если решишь перейти с бесплатной
5. Deploys → Trigger deploy

## Важно про модель
Сейчас по умолчанию используется бесплатная `google/gemma-4-31b-it:free`. Для демо
инвесторам/клиентам это рискованно — бесплатные модели у OpenRouter могут быть медленными
или временно недоступны. Перед серьёзным показом стоит сменить `OPENROUTER_MODEL` на платную
(например любую модель Claude или GPT через OpenRouter) — это займёт одну переменную окружения,
без правок кода.

## Личный кабинет компании (Supabase)

Добавлены: вход/регистрация, роли admin/employee, сохранённые скрипты компании.
Хранилище — Supabase (бесплатный тариф с запасом хватает на пилот).

### Шаг 1. Создай проект Supabase
1. https://supabase.com → New project (бесплатный план)
2. Дождись создания проекта (1-2 минуты)

### Шаг 2. Выполни SQL-схему
1. В проекте: SQL Editor → New query
2. Вставь содержимое файла `supabase-schema.sql` из этой папки → Run
   (создаст таблицы companies/profiles/scripts, политики безопасности и функцию поиска по коду)

### Шаг 3. Отключи подтверждение email (для MVP/пилота)
Authentication → Providers → Email → выключи "Confirm email".
Без этого пользователь после регистрации не сможет сразу войти, пока не подтвердит почту —
для быстрого пилота проще отключить, потом можно включить обратно.

### Шаг 4. Возьми ключи проекта
Project Settings → API → скопируй:
- `Project URL`
- `anon public` ключ

Вставь их в `index.html` в блок:
```js
const SUPABASE_URL = "ВСТАВЬ_SUPABASE_URL";
const SUPABASE_ANON_KEY = "ВСТАВЬ_SUPABASE_ANON_KEY";
```
Это публичные значения, их можно спокойно хранить на фронтенде — доступ к данным
регулируется политиками безопасности (RLS), которые ты применил на Шаге 2.

### Как это работает
- Первый, кто регистрируется через вкладку "Я админ" — создаёт компанию и получает код приглашения
- Этот код раздаётся сотрудникам, они регистрируются через вкладку "Я сотрудник, есть код"
- Админ в своём кабинете добавляет скрипты (с указанием сферы и B2B/B2C) — они сразу появляются
  у всех сотрудников компании в тренажёре как кликабельные карточки над полем скрипта
- Сессия сохраняется между заходами (обновление страницы не разлогинивает)
